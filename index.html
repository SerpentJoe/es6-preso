What is ES6?

  • The ECMAScript version after 5
    • 'use strict';
    • Object.defineProperty
    • JSON.parse
  • ECMAScript is Javascript
  • Blame Microsoft

ES6 contains three types of changes

  • Huge epic changes! (Proxy, generators, Symbols)
  • Minor syntactic polish (Fat arrow, classes, template strings)
  • Not even that (Promise, Reflect, Array/String methods)

Things ES6 does not contain

  • Types
  • Interfaces

Template strings

  • Javascript strings look like 'this' or "this"
  • Now they can also look like `this`!
    • 
var example = `She said "I'm a string" to him`;
  • Why bother?
  • Linebreaks permitted 
    
// Linebreaks in normal strings
var html = '
    \n\
    
    Testing
    \n\

    ';

// Linebreaks in template strings
var html = `
    
    
    Testing
    

    `;

Template strings

  • More importantly, templating 
    
function greet(name) {
    return `Hello, ${name}!`;
}
  • Allows arbitrary operations 
    
function greet(names) {
    return `Hello, ${names.join(', ')}!`;
}

Fat arrow functions

  • 
// Instead of this ...
var greet = function (name) {
    return `Hello, ${name}!`;
};

// ... this
var greet = (name) => {
    return `Hello, ${name}!`;
};

Fat arrow functions

  • Why bother?
  • Javascript functions bind this and arguments
  • Arrow functions don't 
    
// Instead of this ...
var values = keys.map(function (key) {
    return this[key];
}.bind(this));

// ... this
var values = keys.map(key => {
    return this[key];
});

Fat arrow functions

  • Two forms:
  • 
// Block
var values = keys.map(key => {
    return this[key];
});

// Expression
var values = keys.map(key => this[key]);

Destructuring assignment

  • 
// Instead of this

var [name, rank, serialNumber] = values;
  • Same as list in PHP 
    
list($name, $rank, $serialNumber) = $values;

Destructuring assignment

  • Works with objects too 
    
// Instead of this

var rank = this.rank;
var serialNumber = this.serialNumber;

// Do this

  • Similar to extract in PHP 
    
extract($values); // $name, $rank, $serialNumber

let and const

  • var is not good
  • let and const are better 
    
(function () {
    const value = 'outer';
    if (true) {
        let value = 'inner';
        console.log(value); // inner
    }
    console.log(value); // outer
}());
  • Block scoping at last!

let and const

  • Works well with for-of loop

  • No need for _.forEach

Maps and Sets

WeakMaps and WeakSets

  • Don't maintain references for gc purposes
  • Non-enumerable
  • Can't check size
  • Keys must be objects
  • 
var map = new WeakMap;
const obj = {};
map.set(obj, 'first');
map.set({}, 'second');
console.log(map.get(obj)); // first

var set = new Set;
set.add(obj);
console.log(set.has(obj)); // true
console.log(set.has({})); // false

Rest and spread

  • 
// Rest
const [first, second, ...rest] = [1, 2, 3, 4, 5];
console.log(JSON.stringify(rest)); // [3, 4, 5]

// Spread
const foo = [...rest, true, false];
console.log(JSON.stringify(foo)); // [3, 4, 5, true, false]
  • Rest helps with handling varargs 
    
function foo(...args) {
    console.log(args);
}
  • Spread helps with sending varargs 
    
const bar = ['hello', 'world'];
foo(...bar); // ['hello', 'world']

Rest and spread

  • Not in ES6: Object rest and spread
  • 
// Rest
const {hello, ...rest} = {hello:'world', goodnight:'moon'};
console.log(JSON.stringify(rest)); // {"goodnight":"moon"}

// Spread
const oldState = {
    hello: 'world',
    goodnight: 'moon',
};
const newState = {
    ...oldState, // spread
    goodnight: 'and good luck',
};
console.log(newState); // {"hello":"world", "goodnight":"and good luck"}
  • Proposal in ES7

Generators

  • Lazily created sequences 
  • 
function* range(begin, end) {
    for (let curr=begin; curr!==end; curr++) {
        yield curr;
    }
}

for (const curr of range(2, 5)) {
    console.log(curr); // 2, 3, 4
}

Symbols

  • Object keys that never overlap
  • 
const obj = {
    [Symbol('foo')]: 1,
    [Symbol('foo')]: 2,
    [Symbol('foo')]: 3,
};
console.log(Object.getOwnPropertySymbols(obj).length); // 3
console.log(obj[Symbol('foo')]); // undefined
  • The seventh type! 
    
console.log(typeof Symbol('hello world')); // symbol

Proxy

  • Object.defineProperty lets you dynamically respond to property accesses if you've prepared in advance
  • Proxy gives you superpowers 
  • 
const obj = new Proxy({}, {
    set(tgt, prop, value) {
        console.log(`this.${prop} = ${JSON.stringify(value)`);
    },
});

obj.hello = 'world'; // this.hello = "world"
